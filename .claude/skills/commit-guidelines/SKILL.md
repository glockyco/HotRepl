# Commit Message Guidelines

Follow Conventional Commits format with [cbea.ms](https://cbea.ms/git-commit/) prose rules.

## Format

```
type(scope): summary in imperative mood

Prose body explaining why the change was made. The diff shows what
changed — the message explains the reasoning, context, and consequences
that the diff cannot. Wrap at 72 characters.
```

Single-line commits are fine when the change is genuinely self-evident:
```
fix(core): remove off-by-one in subscription frame counter
```

## Types

- **feat** — new capability (protocol message, helper, config option)
- **fix** — bug fix
- **refactor** — restructure without behaviour change
- **test** — add or update tests only
- **docs** — documentation, README, AGENTS.md, skills
- **chore** — build, CI, dependencies, tooling

## Scopes

- `core` — `HotRepl.Core` (engine, evaluator, protocol, server, subscriptions)
- `bepinex` — `HotRepl.BepInEx` adapter
- `protocol` — wire types and serialisation specifically
- `client` — Python reference client and `hotrepl` CLI
- `tests` — xUnit test suite
- `skill` — `.claude/skills/`

## Rules

1. **Subject line = `type(scope): summary`** — imperative mood, no period, ≤ 72 chars
2. **Blank line** between subject and body (required; omitting it breaks `git log --oneline`, `rebase`, `shortlog`)
3. **Body = prose**, 72-char wrap — explain *why*, not *what*; the diff covers *what*
4. **Atomic** — one concept per commit; if you need "and" to describe it, split it
5. **No attribution** — no "Generated by", no co-author footers
6. **Issue refs** at the end of the body: `Part of #N` or `Closes #N`

## Good Example

```
fix(protocol): reject eval requests with empty code field

The server was silently accepting eval requests with an empty code
string and returning a success with no value — indistinguishable from
a legitimate no-op eval. Clients had no way to detect the mistake.
The fix returns eval_error with errorKind "compile" to make the
invalid input explicit.

Closes #17.
```

## Bad Example

```
fix: fixed the bug

- Fixed empty code validation
- Updated MessageRouter
- Added test

🤖 Generated with Claude
```

The bad example lists *what* changed instead of *why*; scope is missing;
summary uses past tense; bullet points replace prose; attribution footer
should not be added.
